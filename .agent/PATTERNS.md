## Framework Structure

```
multi-agent-framework/
├── .agent/                 # Agnostic core (works with any agent tool)
│   ├── agents/             # Role definitions + per-agent state (lead/, architect/, builder/, reviewer/)
│   ├── skills/             # Harness-agnostic reusable agent behaviors
│   │   ├── engineering/    # Bucket: diagnose, tdd, triage, grill-with-docs, zoom-out, …
│   │   │   └── {skill}/    # Each skill: SKILL.md with frontmatter + task prompt
│   │   ├── productivity/   # Bucket: caveman, grill-me, write-a-skill, …
│   │   └── misc/           # Bucket: repo-specific skills (reserved)
│   ├── config/             # Task-system and domain configuration
│   │   └── task-system.md  # How "publish to tracker" maps to MAF task scaffolding
│   ├── context/            # Project-specific context surfaces
│   │   ├── architecture-language.md  # Canonical vocabulary (on-demand, Architect)
│   │   └── project.md      # What this project is
│   ├── models.yaml         # Role→tier mapping (Frontier/Workhorse/Fast)
│   ├── tasks/              # Task execution (active/ and done/)
│   ├── projects/           # Project decomposition (phases, modules)
│   ├── rules/              # Behavioral rules
│   ├── backlog/            # Work queue
│   ├── templates/          # Copyable templates
│   └── handoff/            # Manual handoff documents
├── .opencode/              # OpenCode-specific (agents, plugins, commands)
│   ├── agents/             # Agent definitions (lead, builder, architect, reviewer, builder-upgraded)
│   ├── skills/             # OpenCode skill destinations (generated by maf-skills.sh sync)
│   ├── models.env          # Tier→model-ID assignments (edit this to change models)
│   ├── models.catalog.yaml # Reference catalog of all available models
│   └── plugins/            # Compaction recovery plugin
├── .claude/                # Claude Code-specific
│   ├── agents/             # Claude Code agent definitions
│   └── skills/             # Claude Code skill destinations (generated by maf-skills.sh sync)
├── .cursor/                # Cursor-specific
│   └── rules/              # Cursor skill destinations (generated by maf-skills.sh sync)
├── bin/                    # CLI tools (all maf-*.sh listed below)
└── docs/                   # Design documents and reference
```

## Naming Conventions

| Type | Convention | Example |
|------|-----------|---------|
| Task IDs | kebab-case, descriptive | `auth-token-validation` |
| Work files | `v{N}.md` sequential | `v1.md`, `v2.md` |
| Review files | `r{N}.md` sequential | `r1.md`, `r2.md` |
| Agent state | `STATE.md` per role | `.agent/agents/builder/STATE.md` |
| Project dirs | kebab-case | `.agent/projects/user-portal/` |

## Task Protocol

- [protocol] Tasks live in `.agent/tasks/active/{task-id}/`
- [protocol] Completed tasks archived to `.agent/tasks/done/{task-id}/` (TASK.md + STATUS.md only)
- [protocol] Builder submits work AFTER programmatic gates pass (tests, lint, types)
- [protocol] Reviewer feedback must be specific: location + problem + fix suggestion
- [protocol] Max 3 revision rounds in inner loop, then escalate

## Runtime State vs. Task Artifacts

- `.agent/state/` — **runtime state** (generated by CLIs at execution time; gitignored via per-dir `.gitignore`). Example: `.agent/state/goal/` for `maf-goal.sh` active goal JSON and turn transcripts.
- `.agent/tasks/` — **source-controlled task artifacts** (TASK.md, STATUS.md, work/, reviews/). These are committed and archived.
